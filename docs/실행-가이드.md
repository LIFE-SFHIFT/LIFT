# 놓치면손해 — 로컬 실행 / 종료 가이드

macOS(zsh) 기준. 백엔드(Spring Boot, 8080)와 프론트(Next.js, 3000)를 함께 띄운다.

- 프론트 접속: http://localhost:3000
- 백엔드 API: http://localhost:8080
- Swagger(API 문서): http://localhost:8080/swagger-ui/index.html

> 순서: 백엔드 먼저 → 프론트. 로그인 시 백엔드가 떠 있어야 한다.

---

## 방법 A — 터미널 2개 (권장)

### 터미널 1 · 백엔드 (포트 8080)
```bash
cd /Users/gimgyeong-ug/Documents/LIFT
OAUTH_MOCK_ENABLED=true JWT_SECRET=local-development-jwt-secret-32-bytes-minimum \
  ./gradlew bootRun --args='--spring.profiles.active=local'
```
- `Started ...Application` 로그가 뜨면 준비 완료.
- 실제 OpenAI 응답을 보려면 프로젝트 루트 `.env`에 `OPENAI_ENABLED=true`, `OPENAI_API_KEY=...`를 넣고 위처럼 `local` 프로필로 실행한다.
- `데모용 로그인으로 바로 체험하기`도 백엔드 mock OAuth 토큰을 받아 실제 API를 호출한다. OpenAI 챗봇 테스트는 데모용 로그인, 카카오, 네이버 시작 버튼 중 어느 흐름으로 들어가도 가능하다.
- 끄기: 이 창에서 `Ctrl + C`

### 터미널 2 · 프론트 (포트 3000)
```bash
cd /Users/gimgyeong-ug/Documents/LIFT/frontend
cp -n .env.local.example .env.local   # 최초 1회만
npm install                            # 최초 1회만
npm run dev
```
- 브라우저에서 http://localhost:3000 접속.
- 끄기: 이 창에서 `Ctrl + C`

## 방법 B — 백그라운드로 한 번에 실행

```bash
# 백엔드 시작 (로그: /tmp/lift-back.log)
cd /Users/gimgyeong-ug/Documents/LIFT
OAUTH_MOCK_ENABLED=true JWT_SECRET=local-development-jwt-secret-32-bytes-minimum \
  ./gradlew bootRun --args='--spring.profiles.active=local' > /tmp/lift-back.log 2>&1 &

# 프론트 시작 (로그: /tmp/lift-front.log)
cd /Users/gimgyeong-ug/Documents/LIFT/frontend
npm run dev > /tmp/lift-front.log 2>&1 &
```

상태 / 로그 확인:
```bash
jobs                          # 현재 셸의 백그라운드 작업 목록
tail -f /tmp/lift-back.log    # 백엔드 로그 (Ctrl+C로 로그 보기만 종료)
tail -f /tmp/lift-front.log   # 프론트 로그
```

## 종료 / 연결 끊기

포트로 확실하게 끄기 (방법 A·B 모두 동작):
```bash
lsof -ti tcp:8080 | xargs kill -9    # 백엔드 종료
lsof -ti tcp:3000 | xargs kill -9    # 프론트 종료
```

종료 확인 (아무것도 안 나오면 정상):
```bash
lsof -ti tcp:8080 tcp:3000
```

Gradle 데몬까지 정리(선택):
```bash
cd /Users/gimgyeong-ug/Documents/LIFT && ./gradlew --stop
```

## 참고 (MVP)
- 소셜 로그인 / AI 는 mock 기본값. 결제는 빠른 데모 결제와 토스 테스트 결제 선택지를 함께 제공.
- 토스 테스트 결제 사용 시 프론트 `NEXT_PUBLIC_TOSS_CLIENT_KEY=test_ck_...`, 백엔드 `TOSS_PAYMENTS_ENABLED=true`, `TOSS_SECRET_KEY=test_sk_...` 설정 필요.
- 백엔드 CORS 허용 오리진: `lift.cors.allowed-origins` (기본 `http://localhost:3000`).
